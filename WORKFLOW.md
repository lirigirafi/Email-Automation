# Human-in-the-Loop Workflow Guide

## Understanding the Human-in-the-Loop Process

This agent is designed with **humans first**. Every email requires explicit human approval before sending. This section explains the complete workflow.

## The 5-Step Process

### 1️⃣ Email Fetching
**What happens**: Agent connects to Gmail and retrieves unread emails

```
Agent connects to Gmail API
     ↓
Looks for unread emails
     ↓
Filters by allowed authors (config.json)
     ↓
Skips emails with attachments (flags for manual review)
     ↓
Only new emails (avoids reprocessing)
```

**You control**: 
- Who can be processed (allowedAuthors in config.json)
- How many days back to check (daysToCheck in config.json)

### 2️⃣ Draft Generation
**What happens**: Agent creates professional response drafts

```
Analyzes email content
     ↓
Determines email type (question, acknowledgment, info request)
     ↓
Selects appropriate response template
     ↓
Generates friendly yet professional response
     ↓
Saves draft for your review
```

**You control**:
- Response tone (friendly_professional in config.json)
- Draft format (markdown or JSON)
- Templates used (in src/draft_generator.py)

### 3️⃣ Draft Presentation
**What happens**: Drafts are presented for your review

**Three ways to review:**

**Option A: Markdown Files**
```
data/drafts/draft_20240322_091500.md
```
- Open in any text editor
- View in VS Code with preview
- Edit directly for customization

**Option B: JSON Format**
```
data/drafts/drafts_20240322_091500.json
```
- Structured data format
- Programmatic access
- Perfect for automation

**Option C: Webview (Future)**
- Integrated VS Code panel
- Rich UI for approvals
- Side-by-side comparison

### 4️⃣ Manual Review & Approval
**What happens**: You decide what to send

```
Agent generated draft
     ↓
YOU REVIEW: 
  - Is the tone appropriate?
  - Is the content accurate?
  - Does it address the email?
     ↓
YOU DECIDE:
  ✓ Approve & Send
  ✎ Edit & Customize  
  ✗ Discard & Skip
```

**You control**:
- Every email decision
- Customization before sending
- Discarding inappropriate drafts

### 5️⃣ Sending (With Approval)
**What happens**: Only approved emails are sent

```
You run: python verify_drafts.py [file]
     ↓
See draft 1, 2, 3... one by one
     ↓
For each:
  [y] Send it
  [n] Skip it
  [q] Quit
     ↓
Email is sent directly from your Gmail
```

**You control**:
- Exact moment of sending
- Individual approval per email
- Batch approval option (if confident)

def count_up_to(n):
    for i in range(n):

for x in count_up_to(3):

## Safety Features

### 1. No Automatic Sending
- Agent never sends automatically
- You must explicitly approve each draft
- Even in scheduled mode

### 2. Duplicate Prevention
- Tracks processed emails in `processed_emails.json`
- Prevents generating same draft twice
- Can reset tracking if needed: `python -c "from src.persistence import EmailPersistence; EmailPersistence().reset()"`

### 3. Attachment Flagging
- Emails with attachments skipped
- Flagged for manual review
- Forces human attention to file handling

### 4. Whitelist-Only Processing
- Only processes emails from `allowedAuthors`
- Rejects everything else
- Only you control the author list

### 5. Review-Before-Send
- Every draft reviewed before sending
- Customization always possible
- Individual approval per email

## Decision Tree

```
Email arrives
    ↓
From allowed author? 
    No → Ignored
    Yes ↓
Has attachment?
    Yes → Flagged, skipped
    No ↓
Draft generated
    ↓
You review
    ↓
Approve & Send?
    No → Discarded
    Yes → Edit?
        Yes → Customize, then send
        No → Send as-is
```

## Scheduling Behavior

### Manual Mode
```bash
python src/main.py --mode manual
```
- Runs once
- Generates all new drafts
- Exits
- You manually run `verify_drafts.py`

### Scheduled Mode
```bash
python src/main.py --mode scheduled
```
- Runs at 9 AM and 5 PM (by default)
- Generates drafts automatically
- Waits for your approval
- Keeps running (press Ctrl+C to stop)

### Customizing Schedule
Edit `.env`:
```env
SCHEDULE_TIME_1=08:00  # First run at 8 AM
SCHEDULE_TIME_2=18:00  # Second run at 6 PM
TIMEZONE=America/Chicago  # Your timezone
```

## Approval Checklist

Before sending each draft, verify:

- [ ] **Tone**: Is it appropriate for this recipient?
- [ ] **Grammar**: Any typos or errors?
- [ ] **Relevance**: Does it address their email?
- [ ] **Information**: Did I include all needed details?
- [ ] **Professional**: Would I be proud to send this?
- [ ] **Compliance**: Does it follow org policies?
- [ ] **Personal**: Does it sound like me?

## Performance & Limits

- Process **up to 10 new emails** per run
- Generate drafts in **~100ms per email**
- Track **unlimited processed emails**
- Schedule **2 daily runs** (configurable)
- No rate limits from agent (Gmail API has limits)

## Advanced: Resetting State

### Clear Processed Emails Database
```python
from src.persistence import EmailPersistence
persistence = EmailPersistence('./data/processed_emails.json')
persistence.reset()
```

Creates new database and reprocesses all emails

### Clear Scheduled Jobs
In scheduled mode, press `Ctrl+C` to stop and clear jobs
